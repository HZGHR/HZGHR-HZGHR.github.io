自建图床选择及搭建

互联网工具
工具
发布日期:   2021-12-12
文章字数:   2.3k
阅读时长:   8 分

一、什么是图床

图床，又名图片托管服务，允许个人上传图片到互联网网站。然后图像主机将图像存储到其服务器上，并显示各个不同类型的代码以允许其他人查看该图像。

An image hosting service allows individuals to upload images to an Internet website. The image host will then store the image onto its server, and show the individual different types of code to allow others to view that image.

图床的意义：

  • 减轻服务器压力：图片资源可以由单独的服务器资源来存储。

  • 减轻服务器的存储空间：图片可以托管到其他图片托管平台。

  • 提供CDN加速，没有容量限制

二、为什么选择自建图床

图床服务主要用途还是在建站方面，如果选择第三方免费图床，会面临服务商倒闭、防盗链限制、图片丢失、数据被删除等意外情况。例如很早以前网友们用新浪微博的图床服务，但自从新浪微博防盗链，不让外链调用使用后，图床服务就挂掉了。一些境外的免费图床也时常面临IP被墙，加载太慢的情况。因此，自建图床对于建站稳定很重要。

三、自建图床服务选择

（一）OneDrive、Google drive等网盘

OneDrive和Google drive是微软和谷歌的网盘服务，使用OneDrive做博客的图床，是很多个人站长的的选择。相对于其它图床，有以下几点优势：

  1. 便宜。用Office365家庭版，209-229元即可买到一年。6个用户平均下来，每人每年只要35-38元。
  2. 容量大。每人有1TB的空间，做博客图床绰绰有余，可以说永远不会不够用。
  3. 官方支持外链和https，且不限流。比起各种奇技淫巧的方法，比如“微博图床”，只要运营商想，分分钟可以关掉外链。而一些限流的免费图床，如果以后博客做大，流量增加，巨大的迁移成本，会让你不得不选择缴纳高昂的费用。
  4. 微软大公司背书。一些国内图床（比如七牛云），虽然倒闭/转型/被勒令关闭的可能性也不大，但终究有这种可能。而对于承载了博客所有图片资源的图床来说，简直是致命的。相比起来，首富家的微软产品，特别是同时为企业服务的OneDrive，这种可能性基本为0。
  5. 国内可用。比起GitHub这类虽然免费，但国内几乎无法访问的图床，OneDrive尽管也有不稳定的时候，但大多情况下依然是可以访问的。

但这种方式是利用OneDrive API接口来上传图像或者获取直链，但微软azure中国区服务被各种限制，门户网站登录经常失败，而且微软限制了API接口调用次数上限，调用频繁会被限制访问，因此，该方式并不是长久之计。但OneDrive可以搭建低频访问的下载站（分享网盘）。

（二）私人服务器

一些大型图片类的网站为了减轻带宽压力，用户会选择单独租用服务器来放置外链图片，特别是漫画网站和一些电子商务网站对这类对带宽大小和质量都要求高的网站，租用VPS配置图床服务器就非常有必要。

图床服务器VPS配置要求：由于图床本身也是一种存储业务，对CPU的要求虽然不高，所以租用VPS已经足够应用了。因为图床更多的是对图片数据的安全和链接管理数据库的稳定性有更多的要求，所以对于内存和硬盘的需求要充足。
（1）建议选择SSD作为系统盘，大容量企业级混合硬盘作为存储磁盘可以保证足够的存储容量;
（2）从线路方面来看网站的要求，免备案的香港、台湾服务器的直连线路比较合适，保证图快速加载，无需备案，搭建便捷，在价格方面相也能节省成本。
（3）先择更专业的机房合作，在选择图床服务器机房时，建议选择能够提供24小时在线服务的专业机房，一旦出现故障，可能会导致主网站上的图片丢失。

但对于绝大多数小站长来说，租用高速线路VPS的成本太高，并不划算。

（三）Github、Gitee等公共仓库

Github、Gitee都是代码托管平台，每个仓库有300M的储存容量，更重要的是它们是免费的。首先在Github创建公开的仓库，上传图片到仓库里，获取图片直链。除此之外，还可以利用CDN加速访问，例如免费的JsDelivr。

但Github服务器在境外，加上容量有限，所以不能作为长久之计。Gitee作为国内的代码托管平台，也有容量限制，但它服务器在境内，可以不用加速，也可以快速访问。如果想利用公共仓库做图床，可以考虑Gitee和Code等国内代码托管平台。

（四）OSS对象储存服务

对象存储（Object Storage Service）是面向海量非结构化数据的通用数据存储平台，提供安全可靠、低成本的云端存储服务。作为数据存储和流转中心，可通过浏览器、HTTP RESTful API 、S3 API 、 SDK 和 FTP 等方式高效存取和管理文件，支撑企业丰富的上层业务和数据分析系统使用。

Object storage (also known as object-based storage) is a computer data storage architecture that manages data as objects, as opposed to other storage architectures like file systems which manages data as a file hierarchy, and block storage which manages data as blocks within sectors and tracks. Each object typically includes the data itself, a variable amount of metadata, and a globally unique identifier. Object storage can be implemented at multiple levels, including the device level (object-storage device), the system level, and the interface level. In each case, object storage seeks to enable capabilities not addressed by other storage architectures, like interfaces that are directly programmable by the application, a namespace that can span multiple instances of physical hardware, and data-management functions like data replication and data distribution at object-level granularity.

四、又拍云图床搭建

  1. 首先注册又拍云账号：https://console.upyun.com/register/?invite=H1PJesVEL
  2. 实名认证，不认证的话，使用不了存储业务和CDN加速
  3. 在又拍云产品中心，选择云储存，创建云储存服务
  4. 授权操作员，用于第三方插件上传和删除图片
  1. 绑定自己的域名，并配置CNAME
  2. 创建目录，设置储存路径
  3. 设置HTTPS访问，添加SSL证书，设置防盗链、访问加速等

至此，基于又拍云OSS服务的图床基本设置就完成了。

五、PicGo图床上传设置

但是为了方便写作中快速插入，需要使用第三方插件自动上传图片并生成Markdown语法的图片直链，这里我使用的是PicGo

（一）PicGo客户端+Typora

  1. 下载PicGo客户端：https://github.com/Molunerfinn/PicGo/releases
  2. 安装好客户端后，配置图床
    这里对照上一步又拍云的图床设置来填就好，注意操作员及授权密码正确，填写储存路径以 / 结尾，填写完毕后设置为默认图床。
  3. 设置上传重命名，上传后自动复制URL等
  4. 打开Typora偏好设置，图像设置，将上传服务选择为PicGo App，设置绝对路径，并将插入图片时的默认操作设置为上传图片。这样在使用Typora进行写作时，直接将图片复制粘贴到文章里就可以了。

（二）Vscode+PicGo插件

  1. 下载安装vscode
  2. 在vscode插件市场下载安装以下插件:
    • Chinese（Simplified（简体中文）
    • Markdown all in One
    • Markdown Preview Enhanced
    • Markdown Preview Github Styling
    • PicGo
  3. 打开插件PicGo拓展设置，将Picgo Pic Bed: Current和Picgo Pic Bed: Uploader都设置为 upyun,在upyun的参数设置中，填入服务名称、操作员、授权密码、储存路径、绑定的域名。
  4. 此后，就可以使用快捷键来自动上传图片并自动插入到文章中了
  • 从剪贴板上传图像：Ctrl + Alt + U
  • 从资源管理器上传图像：Ctrl + Alt + O
  • 从输入框上传图像 ：Ctrl + Alt + E

经过综合对比，typora上传图片容易出现图片加载失败的情况，加上typora改为收费订阅制后，更建议使用vscode进行markdown编写。但vscode相比于typora，需要使用者更加熟悉markdown语法进行手动输入，而typora则集成了一些快速功能，对小白更友好。熟练掌握markdown语法后，还是建议使用vscode。
